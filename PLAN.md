# 恶搞启动器生成工具开发计划

## 1. 项目概述
本项目旨在将原有的单一脚本改造成一个通用的“恶搞启动器生成工具”。该工具包含两个部分：
1.  **生成器 (Builder)**：供恶搞方使用，提供可视化界面，用于配置并一键生成恶搞 exe。
2.  **启动器 (Launcher)**：被恶搞方运行的程序，具备伪装界面、全盘扫描、自动启动或跳转网页的功能。

## 2. 功能模块设计

### 2.1 生成器 (Builder) - 恶搞方工具
*   **可视化配置界面**：
    *   **目标程序设置**：
        *   目标进程名 (如 `YuanShen.exe`)。
        *   目标程序描述 (如 `原神`)。
        *   失败跳转网址 (如 `https://ys.mihoyo.com`)。
    *   **伪装设置**：
        *   生成文件名 (如 `植物大战僵尸3.exe`)。
        *   自定义图标 (选择 `.ico` 文件)。
        *   **文件体积膨胀**：设置目标文件大小 (如 200MB)，通过填充垃圾数据增加真实感。
*   **一键打包核心**：
    *   基于用户配置动态生成启动脚本。
    *   调用 PyInstaller 进行独立 exe 打包。
    *   执行文件体积膨胀操作。

### 2.2 启动器 (Launcher) - 被恶搞方产物
*   **伪装加载界面**：
    *   启动后显示全屏或窗口化的“正在加载资源...”进度条。
    *   模拟游戏/软件的初始化过程，迷惑用户。
*   **核心逻辑**：
    *   **全盘扫描**：在后台线程静默扫描目标程序 (`xxx.exe`)。
    *   **寻找日志**：可选显示或隐藏扫描过程日志。
    *   **自动执行**：
        *   **Found**: 找到目标程序 -> 关闭伪装界面 -> 启动目标程序。
        *   **Not Found**: 扫描完成未找到 -> 关闭伪装界面 -> 自动打开浏览器跳转预设网址。

## 3. 技术实现方案

### 3.1 目录结构规划
```
project_root/
├── builder/                # 生成器相关代码
│   ├── builder_gui.py      # 生成器可视化界面 (Tkinter)
│   └── file_inflator.py    # 文件膨胀工具
├── template/               # 启动器模板代码
│   ├── launcher_core.py    # 扫描与启动逻辑
│   ├── fake_ui.py          # 伪装界面 (Tkinter)
│   └── boot.py             # 启动入口 (将被动态注入配置)
├── assets/                 # 默认资源 (默认图标等)
└── output/                 # 生成的 exe 输出目录
```

### 3.2 关键技术点
1.  **配置注入**：
    *   生成器将用户配置（目标exe名、网址等）以字典形式写入临时 Python 文件，作为启动器的配置模块。
2.  **文件膨胀 (File Inflation)**：
    *   在 PyInstaller 打包完成后，以二进制追加模式 (`ab`) 向 exe 文件末尾写入 0 字节，直到达到指定大小。Windows PE 文件格式允许尾部追加数据而不影响运行。
3.  **全盘扫描优化**：
    *   使用多线程进行扫描，避免阻塞 UI 线程（防止界面卡死）。
    *   优先扫描常见安装目录（注册表、Program Files 等）。
